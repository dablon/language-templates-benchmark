Project Scaffolding CLI

Generate production-ready web service projects in Go, Python, Rust, and C with a single command

🚀 Quick Start

# Download the CLI for your platform
curl -L https://github.com/dablon/language-templates-benchmark/releases/latest | grep -o 'scaffold-cli-.*'

Or build from source:

git clone https://github.com/dablon/language-templates-benchmark.git
cd language-templates-benchmark/scaffold-cli
cargo build --release
./target/release/scaffold

📋 CLI Usage

scaffold [OPTIONS]

Options:
  -n, --name <NAME>          Project name (required)
  -l, --language <LANG>      go, python, rust, c [required]
  -p, --protocol <PROTO>     http, grpc, service-mesh [required]
  -d, --database <DB>        none, postgres, postgres-redis [default: none]
  -o, --output <DIR>         Output directory [default: .]
  -h, --help                  Show help

💡 Examples

Create a Go HTTP API

scaffold -n my-api -l go -p http -d none -o ./projects
HTTP No DB

Create a Rust gRPC service with PostgreSQL

scaffold -n my-grpc -l rust -p grpc -d postgres -o ./projects
gRPC PostgreSQL

Create a Python Service Mesh project

scaffold -n my-mesh -l python -p service-mesh -d postgres-redis -o ./projects
Service Mesh Consul PostgreSQL + Redis

🔧 Protocols

HTTP

Pure REST API with JSON. No gRPC or service mesh dependencies. Fastest for simple microservices.

gRPC

High-performance RPC using Protocol Buffers. Ideal for internal service communication.

Service Mesh

HTTP + Consul service discovery. Full mesh integration with health checking.

🗄️ Database Options

None

No database. Pure compute/microservice without persistence.

PostgreSQL

Single database connection with connection pooling.

PostgreSQL + Redis

Full stack with cache layer for high-performance apps.

📁 Generated Project Structure

my-project/
├── src/                  # Language-specific source
│   └── main.{rs,go,py,c}  # Entry point
├── static/              # Static files (HTML, CSS, JS)
├── Dockerfile            # Multi-stage container build
├── docker-compose.yml    # Full stack with optional DB
├── .env.example         # Environment template
└── README.md            # Project docs

# API Endpoints (all templates)
GET  /           → Home page
GET  /health     → Health check
GET  /api/hello  → JSON greeting
GET  /api/compute → CPU benchmark
POST /api/echo    → Echo body

🐳 Docker Support

Every generated project includes Docker support:

# Build and run
docker compose up --build

# Run specific service
docker compose up -d go-http

# Scale services
docker compose up --scale go-http=3
